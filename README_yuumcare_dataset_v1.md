# YuumCare Anonymized Screening Dataset

## Dataset title
YuumCare anonymized screening data for early detection and decision support in higher education.

## Related manuscript
**A Scalable Data Pipeline for Early Detection and Decision Support in Higher Education: YuumCare**

## Description
This repository contains an anonymized version of the transformed analytical dataset used to reproduce the descriptive results reported in the YuumCare manuscript. The dataset corresponds to a non-clinical screening initiative with first-year university students and is intended for transparency, reproducibility, and academic reuse.

The public dataset contains **466 records** and **28 variables**. It includes demographic groupings, ordinal indicators, binary indicators, multi-hot encoded support needs, and a derived non-clinical risk classification label.

## Files
- `yuumcare_anonymized_screening_data_v1.csv`: anonymized analytical dataset.
- `yuumcare_data_dictionary_v1.csv`: variable-level data dictionary.
- `yuumcare_preprocessing_reproducibility_script_v1.py`: simplified script to validate and reproduce main descriptive outputs from the anonymized dataset.
- `README.md`: description of the dataset package.

## Anonymization and privacy
The dataset was prepared for public academic release after removing direct personal identifiers. The following privacy-preserving steps were applied:
- Removal of names, student IDs, contact details, program identifiers, and free-text responses.
- Assignment of synthetic respondent identifiers.
- Age grouping into broad intervals.
- Use of structured categorical, ordinal, binary, and multi-hot encoded variables.
- Explicit preservation of selected non-responses as `No response` rather than imputation or record deletion.

No personally identifiable information is included in this public dataset.

## Variable structure
The dataset reflects the structured data pipeline described in the manuscript:
1. Data ingestion and consolidation.
2. Data preparation and anonymization.
3. Feature construction using ordinal encoding, binary mapping, discretized age ranges, and multi-hot encoding.
4. Rule-based non-clinical risk classification.

Some ordinal indicators use simplified ordinal schemes depending on the structure of the original questionnaire item. Therefore, ordinal ranges may differ across variables while preserving relative order.

## Missing data
Missing responses for selected support-related indicators are explicitly encoded as `No response`. This is intentional and supports transparency in reproducing the descriptive analyses.

## Reproducibility
The included Python script validates the dataset and reproduces key descriptive outputs, including:
- Sample size.
- Age range distribution.
- Gender distribution.
- Help-seeking interest distribution.
- Selected `No response` counts.
- Support-need prevalence.
- Risk-level distribution.
- Wilson confidence intervals for selected prevalence estimates.
